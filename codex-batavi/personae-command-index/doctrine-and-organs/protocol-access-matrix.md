# Protocol Access Matrix — Axiom Lupus custody lattice

*Sealed governance file. Read with `contingency-axiom-lupus.md`.*

This matrix defines duty-based access boundaries for anti-inversion protocol content.

---

## 1. Access doctrine

- Compartmentalization follows duty necessity, not prestige.
- Each tier receives sufficient data for its mission and no more.
- Any expansion of access must be logged and justified by trigger state.

---

## 2. Tier map

## Tier A — full strategic custody

Holders:

- Varro
- Drusus
- Tyvar
- senior high-command Legatus set
- Radulf (intervention requirement basis)
- Guilliman (request-level sovereign access)

Authorized content:

- full trigger ladder and thresholds
- full cohort sequencing and fallback maps
- command firewall architecture
- cull authorization gates
- post-event rollback and continuity substitutions

## Tier B — limited operational custody

Holders:

- Kadmos
- Otho

Authorized content:

- activation signals and handshake protocols
- execution geometry for assigned phases
- immediate objective envelopes by trigger tier
- minimum legal frame required for compliant action

Denied content:

- full strategic rationale tree
- full cross-tier custody map
- non-essential long-horizon contingency branches

## Tier C — objective-only awareness

Holder:

- Alaric

Authorized content:

- protocol existence
- mission objective (chapter + Imperium continuity)
- duty basis for compartmentalization

Denied content:

- kill-route geometry
- detailed executor choreography
- matrix internals and fallback architecture

---

## 3. Governance events

## A. Lock event

- Establish baseline holders and archive hash.
- Confirm signatures for all Tier A holders.

## B. Escalation event

- Temporary elevation of specific Tier B holders during Tier II or III triggers.
- Elevation requires dual authorization from Varro and Drusus.

## C. Rollback event

- Mandatory reversion to baseline custody after stabilization window.
- Post-event review required before closing.

## D. Audit event

- Access log verification by purpose, window, and outcome.
- Unauthorized diffusion is treated as doctrine breach.

---

## 4. Continuity clauses

- If one Tier A office is unavailable, substitution chain must keep full-custody coverage.
- No single casualty may collapse protocol authority.
- Tier B cannot self-expand scope beyond signed assignment envelope.

### Tier A Legatus substitution map

| Vacant seat | First substitute | Second substitute |
|-------------|------------------|-------------------|
| *Prima* | *Secunda* | *Quinta* |
| *Quinta* | *Prima* | *Tertia* |
| Other I–IX | Theater-linked Legatus (*Strategium* map) | Radulf (witness until re-seat) |

Substitute may not execute kill-route phases without **Drusus** briefing certification. Partial briefing → Tier I–II authority ceiling only.

### Triumvirate partial availability

- **Varro unavailable:** Drusus co-seal required for matrix rewrite; Otho holds Blindgate continuity.
- **Drusus unavailable:** Clinical lock frozen at last hash; Varro + Tyvar may execute Tier II firewall; Tier III requires dual Tier A Legatus confirm + Tyvar readiness.
- **Both unavailable:** Tyvar + Radulf + Corbec convoke emergency Ring; Tier B executors only until Tier A reconstituted.

## 5. Disclosure control rules

- Disclosure above objective-only level to Alaric requires exceptional dual authorization and documented reason.
- Routine operations cannot be used to normalize deeper disclosure.
- Any breach of disclosure boundary triggers immediate matrix audit.
